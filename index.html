Git

Konsep Git :

  1. Working tree : folder tempat kita bekerja atau melakukan coding
  2. Stagging area : menyiapakan perubahan file atau folder yang kita inginkan pada github
  3. History area : menyimpan dengan cara commit untuk memasukkan pada github
  4. fast forward : menggabungkan cabang yang kita buat dengan jalur master
  5. Three way Merge : menggabung kan cabang yang kita buat dengan cabang lain yang kita buat yang lain
  6. Conflict Merge : penggabungan repo terjadi masalah dikarenakan branch memiliki perubahan yang tidak sama pada file dengan baris yang sama
  7. Origin : nama default remote

Command Git :

  • git init : untuk membuat folder local pada laptob menjadi repo atau folder master
  • git status : untuk melihat status perubahan atau penambahan file atau folder pada folder master
  • git rm --cached (file)... : untuk menghapus file pada stagging area
  • git config --list : melihat konfigurasi pada git kita
  • git config --global user.name "rindu aji" : mensetting git dengan github yang kita inginkan dengan memasukkan nama untuk memasukan perubahan pada github
  • git config --global user.email "rinduaji1994@gmail.com" : mensetting git dengan github yang kita inginkan dengan memasukkan email untuk memasukan perubahan pada github
  • git commit -m "(penjelasan perubahan)" : menyimpan perubahan pada folder ke stagging area
  • git commit -am "(penjelasan perubahan)" : menyimpan semua perubahan pada folder ke stagging area
  • git add . : memasukkan semua file atau folde perubahan pada stagging area
  • git log : melihat semua history commit dan kode hash history pada github secara git. mengakhiri dengan menekan huruf q.
  • git log -3 : melihat 3 history commit dan kode hash history terakhir pada github secara git. mengakhiri dengan menekan huruf z atau q.
  • git log -- (nama file) : melihat history commit dan kode hash history pada file yang kita cari pada github secara git. mengakhiri dengan menekan huruf z atau q.
  • git checkout (kode hash) : mengembalikan folder master pada commit yang diinginkan dengan memasukkan kode hash
  • git branch : mengecek branch yang terdaftar pada repo
  • git branch (nama branch) : membuat branch atau cabang baru pada repo
  • git checkout (nama branch) : merubah lokasi branch pada repo
  • git log --all --decorate --oneline --graph : melihat grafik atau jalur pada repo
  • alias (nama variabel)="(command git)" : merubah command default git menjadi command lain yang kita inginkan agar command tidak terlalu panjang
  • git merge (nama branch) : menggabungkan branch yang diingkan dengan jalur master
  • git branch -d (nama branch) : menghapus cabang pada repo
  • git branch -D (nama branch) : menghapus branch beserta file yang yang terrecord pada branch tersebut
  • git branch --merged : melihat daftar branch yang telah digabungkan
  • git clone (link github) : menyimpan repo pada github ke local komputer kita
  • git remote : melihat list yang remote terdaftar pada repo
  • git remote -v : melihat list yang remote terdaftar pada repo dengan lebih detail
  • git push : mengirimkan atau menyimpan repo pada local komputer kita ke github
